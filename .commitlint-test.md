# Commitlint Test Results

## Test Cases

### Invalid Commits (Should Fail)

- `Added new feature` - Missing type and colon
- `feat add feature` - Missing colon
- `Feat: new feature` - Type should be lowercase
- `feat: a` - Subject too short
- `unknown: new feature` - Invalid type
- `feat: new feature.` - Subject ends with period

### Valid Commits (Should Pass)

- `feat: add new feature`
- `feat(web): add deposit modal`
- `fix(api): resolve JWT expiration`
- `docs: update README`
- `chore: update dependencies`
